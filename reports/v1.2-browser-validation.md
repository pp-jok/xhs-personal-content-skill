## 6. Media Handling

- Image remote URLs are redacted as `<redacted>`.
- Signed or tokenized media URLs are not written to this report.
- Hydration-state image references are marked `skipped_sensitive_url`.
- Video media is recorded as structural evidence from hydration state with `has_media=true`; signed media URL values are not persisted in report text.
- v1.2 does not perform OCR, transcription, keyframe extraction, or video semantic analysis.

## 8. Unstable Or Partial Fields

- Current visible comments were not extracted from either live page.
- The live pages store the main note data in hydration state; DOM-only selectors are insufficient for current Xiaohongshu pages.
- Media file download is intentionally skipped when the source is a signed or sensitive URL.
- The extractor may need additional strategies if Xiaohongshu changes `noteDetailMap` shape.

## 9. Capability Status

- Real implementation:
  - Connect to user-authorized Chrome through CDP.
  - Capture a single user-provided link.
  - Read live page HTML through browser context.
  - Extract title, body, author, publish time, metrics, image references, video evidence, HTML snapshot, screenshot, and diagnostics.
- Partial implementation:
  - Live comments are not reliably extracted.
  - Image/video media files are not downloaded when URLs are sensitive; references are redacted.
- Fallback available:
  - `--manual-file` remains available and is marked `capture_method=manual`.
- Not implemented:
  - OCR.
  - Audio transcription.
  - Video keyframes.
  - Media semantic analysis.
  - Batch crawling.
  - Access-control bypass.
